Formularios

  • form: Sera el elemento padre que anide los campos del formulario
  • --> action: Indica la URL que se enviara a la peticion HTTP
  • --> method: Indica si la peticion HTTP sea GET o POST
  • input: Permite introducir diferentes ttipos de campo de formulario en base al valor del atributo type. En funcion del valor indicado del type diispondremos de unos u ptros atributos
  • --> type:(obligatorio) Puede tener text, email, checkbox, color, dat, file, hidden, etc. En funcion del tipo que queramos
  • --> id: Obligatorio si el elemento label tiene un atributo for
  • --> name: Opcional Representa el nombre asignado al campo cuando se envie la peticion HTTP
  • --> value: Opcional Representa el valor que se asignara al campo cuando se envie la peticion HTTP
  • select: Nos permite crear una lista desplegable de opciones
  • --> id: Igual que el elemento input
  • --> name: Igual que el campo input
  • option: Nos sirve para encapsular cada opcion de la lista
  • -->value: Igual que el atributo value del campo input
  • textarea:representa un campo que nos permite introducir textos con saltos de linea incluidos, normalmente se usa cuando hay que introducir: descripciones, biografias, etc.
  • --> id: Igual que el elemento input y select
  • --> name Igual que el elemento input y select
  • label:Se usa para especificar la etiqueta (o nombre) del campo (input, select o textarea) al que hace referencia la etiqueta.
  • --> for: Debe tener el mismo valor que el atributo id del campo(input, select o textarea) a que hace referencia la etiqueta
  • button: Representa un boton y el texto del boton esta representado por su contenido.
  • --> type: Define el comportamiento del boton cuando esta activadi y ouede contener tres valores: submit, reset, button
